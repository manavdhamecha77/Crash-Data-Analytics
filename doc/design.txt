Design language
Aesthetic direction: warm editorial minimalism. Think a financial terminal crossed with a
high-end print publication. Calm, authoritative, uncluttered.
The one rule to remember: let whitespace do the work. Fewer elements, more breathing room,
higher trust.

Layout rules
Page wrapper
Every page (page.js) renders its own content with no surrounding shell — the shell is in
layout.js. Use:
jsx<div style={{ display:'flex', flexDirection:'column', gap:'28px', maxWidth:'860px', margin:'0 auto' }}>
The layout already adds paddingTop: 96px, side padding, and centers the max-w-5xl container.
Standard page structure (top to bottom)

Topbar row — page title (left) + live chip + clock + primary CTA button (right)
Warning banner (conditional, only when there's an active alert)
KPI row — 4 metric cards in a CSS grid
Two-column content — grid-template-columns: 1fr 280px — main content left, sidebar right
Agent log / secondary content — below the two-col if needed

What NOT to do

No dark backgrounds, no neon glows, no heavy drop shadows
No more than two accent colors visible on screen at once
No emoji in UI (use inline SVG icons instead)
No font-weight above 500 in body content
No ALL CAPS except section titles (11 px, tracked, muted)
No sidebar, topnav, or branding in page files — the layout handles all of that
No spacing tighter than 6 px between interactive elements
Never use position: fixed inside page content (breaks Next.js layout scroll)
